Welcome to the Ultimate Git(hub) Cheatsheet

Git und Github sind nicht das gleiche! Git ist ein VSC-Tool und GitHub ist eine Plattform die Git benutzt, es gibt auch andere zb. GitLab, GitBucket, BitBucket, Sourceforge und so weiter.

Was ist Git?

Git wird überall benutzt und ist eins der wichtigsten Tools für Entwickler geworden. Dadurch können Entwickler von überall zusammenarbeiten.

Wo wird Git benutzt?

Git ist ein Versionverwaltungstool, das bei jedem comitten eine Art Schnappschuss von deinem Projekt macht. Hinterher hat man einen Stapel von Schnappschüssen.

Was macht Git?

initialisiert ein lokales git repository

git init

gibt an was alles in den commit rein soll ( . gibt an das alles in den commit sol)

git add .

hinterlegt einen message/commit ( man kann auch sagen erlegt eine version an)

git commit -m “message”

schickt das lokale repository in das github repository

git push

zeigt alle die bis dato erstellten commits ausführlich an

git log

zeigt alle die bis dato erstellten commits in kurzform an

git log --oneline

Erstellt einen neuen Branch

git branch “name”

Zeigt alle erstellten Branches an

git push

Löscht den ausgewählten Branch

git branch -d “name”

Wechseln zwischen den Branches

git checkout

Lädt die aktuelle Version des Repository von GitHub runter

git pull

Kopiert ein fremdes Projekt von Github

git clone

Ist dafür da um zwei branches zusammenzuführen

git merge
A Added (This is a new file that has been added to the repository)
M Modified (An existing file has been changed)
D Deleted (a file has been deleted)
U Untracked (The file is new or has been changed but has not been added to the repository yet)
C Conflict (There is a conflict in the file)
R Renamed (The file has been renamed)
S Submodule (In repository exists another subrepository)

Lokal = Working Directory → Änderungen werden als Modified dargestellt

Staged Area = Index → Daten werden für den commit vorgemerkt sind aber noch nicht hochgeladen

.git directory repository → Daten sind von lokal auf git geklont worden und stehen als snapshot bereit
